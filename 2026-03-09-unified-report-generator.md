# 统一报告生成器使用说明

_2026-03-09_

## 🎯 核心功能

**自动生成标准格式的分析报告**，包含：
1. **统计摘要** - 一目了然的数据统计
2. **文件清单** - 所有输出文件列表
3. **重点文件说明** - 关键文件位置
4. **下一步建议** - 明确的后续操作指南

---

## 📊 报告模板

### 1. 完整流程报告（4 个阶段）

**标题**: 完整 SRC 信息收集报告

**统计摘要**：
```
### 阶段 1: 子域名收集
- **发现**: 36 个子域名

### 阶段 2: 服务探测
- **可访问**: 10 个 URL

### 阶段 3: 端口扫描
- **开放端口**: 50+ 个

### 阶段 4: 深度分析
- **高风险问题**: 3 个
```

**文件清单**：
```
- stage4/final_report.md - 最终深度分析报告
- stage2/http_accessible_urls.txt - 可访问的 URL 列表
- stage3/web_ips.txt - Web 服务 IP 列表
- stage3/all_accessible_urls.txt - 所有可访问的 URL（汇总）
```

**下一步建议**：
```
1. 查看最终报告
2. 测试可访问的 URL
3. 进行漏洞验证
```

---

### 2. 快速分析报告（单 URL）

**标题**: 快速深度分析报告

**统计摘要**：
```
### 输入 URL
- https://jxgj.jshbank.com/wxcp/index.html

### 分析结果
- 标题: 智薪管家
- 类型: 微信小程序/Web 应用
- 状态: 可访问（200 OK）
```

**文件清单**：
```
- REPORT.md - 快速分析报告
- urls.txt - 输入的 URL
```

**下一步建议**：
```
1. 下载并分析 JS 文件
2. 枚举 Vue.js 路由
3. 路径爆破测试
4. API 端点测试
```

---

### 3. 端口扫描报告

**标题**: 端口扫描报告

**统计摘要**：
```
### 端口扫描结果
- **开放端口**: 50+ 个
- **Web 服务 IP**: 10 个
```

**文件清单**：
```
- port_scan.gnmap - 端口扫描结果
- web_ips.txt - Web 服务 IP 列表
- all_ips_unique.txt - 所有发现的 IP
```

**下一步建议**：
```
1. Web 服务扫描
2. 端口服务分析
3. 继续完整流程
```

## 🚀 使用方法

### 在脚本末尾调用

```bash
#!/bin/bash

# ... 分析代码 ...

# 在末尾添加：
source utils/generate_report.sh

# 生成报告
bash utils/generate_report.sh \
  "${OUTPUT_DIR}" \
  "${TARGET}" \
  "full_stage"  # 或 "quick_analysis" 或 "port_scan" 或 "service_scan"
```
