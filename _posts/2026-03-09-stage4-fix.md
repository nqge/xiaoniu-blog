# 阶段 4 深度分析修复 - 确保所有分析都能正确执行

_2026-03-09_

## 🎯 问题发现

**用户反馈**：
> "优化脚本解决阶段 4 未执行的问题"

**问题分析**：
- 旧版本的自动化脚本在阶段 4 深度分析部分存在输出路径问题
- JS 文件分析、Vue.js 检测、Actuator 检测、路径爆破测试的结果没有正确保存
- 导致深度分析阶段"空转"，没有实际产出

---

## 🔧 核心修复

### **1. JS 文件分析（阶段 4）**

**问题**：
- `jsfind.py` 工具接受文件输入，但脚本传入的是单个 URL
- 结果保存到当前目录，而不是 `${OUTPUT_DIR}`

**修复**：
```bash
# 创建临时 URL 文件
> "${OUTPUT_DIR}/stage4/js_urls.txt"

# 只保存 200 OK 的 URL
for url in $(cat "${OUTPUT_DIR}/stage2/http_accessible_urls.txt"); do
    STATUS=$(curl -Isk "$url" | awk '{print $2}')
    if [ "$STATUS" = "200" ]; then
        echo "$url" >> "${OUTPUT_DIR}/stage4/js_urls.txt"
    fi
done

# 使用 jsfind.py 分析文件
python3 core/jsfind.py "${OUTPUT_DIR}/stage4/js_urls.txt" "${OUTPUT_DIR}/jsfind_results"
```

**效果**：
- ✅ 只分析 200 OK 的 URL（提高效率）
- ✅ 结果正确保存到 `jsfind_results/` 目录
- ✅ 生成 `api_endpoints.txt`、`paths.txt`、`secrets.txt`

---

### **2. Vue.js 检测（阶段 5）**

**问题**：
- 重复扫描 HTTP 服务（效率低）
- 结果没有合并保存

**修复**：
```bash
# 使用 js_urls.txt，避免重复扫描
for url in $(cat "${OUTPUT_DIR}/stage4/js_urls.txt"); do
    python3 core/vuecrack.py "$url" "${OUTPUT_DIR}/stage4/vuecrack_$(basename $url).txt"
done

# 合并结果
cat ${OUTPUT_DIR}/stage4/vuecrack_*.txt > "${OUTPUT_DIR}/stage4/vuecrack_combined.txt"

# 统计
VUE_COUNT=$(grep -c "Vue.js" "${OUTPUT_DIR}/stage4/vuecrack_combined.txt")
echo "[+] 检测到 $VUE_COUNT 个 Vue.js 应用"
```

**效果**：
- ✅ 避免重复扫描（提高效率）
- ✅ 结果合并保存到 `vuecrack_combined.txt`
- ✅ 统计检测到的 Vue.js 应用数量

### **4. 路径爆破测试（阶段 7）**

**问题**：
- 只使用 JS 提取的路径（字典太小）
- 结果没有正确合并

**修复**：
```bash
# 1. 从 JS 文件提取路径
if [ -f "${OUTPUT_DIR}/jsfind_results/paths.txt" ]; then
    cp "${OUTPUT_DIR}/jsfind_results/paths.txt" "${OUTPUT_DIR}/stage4/extracted_paths.txt"
fi

# 2. 添加常见管理路径
cat >> "${OUTPUT_DIR}/stage4/extracted_paths.txt" << 'EOF'
/admin
/admin/login
/admin/dashboard
/api
/api/v1
/api/v2
/config
/settings
/management
/console
/control
/actuator
/actuator/health
/actuator/env
/env
/health
/info
/beans
/mappings
/heapdump
/threaddump
EOF

# 3. 测试路径
for url in $(cat "${OUTPUT_DIR}/stage4/js_urls.txt"); do
    python3 core/path_bruteforcer.py "$url" \
        "${OUTPUT_DIR}/stage4/extracted_paths.txt" \
        "${OUTPUT_DIR}/stage4/path_bruteforce_$(basename $url).txt"
done

# 4. 合并结果
cat ${OUTPUT_DIR}/stage4/path_bruteforce_*.txt > "${OUTPUT_DIR}/stage4/path_bruteforce_combined.txt"

# 5. 统计
ACCESSIBLE_COUNT=$(grep -c "^- \[200\]" "${OUTPUT_DIR}/stage4/path_bruteforce_combined.txt")
echo "[+] 发现 $ACCESSIBLE_COUNT 个可访问路径"
```

**效果**：
- ✅ 结合 JS 提取的路径 + 常见管理路径（更全面）
- ✅ 只测试 200/403 的 URL（提高效率）
- ✅ 结果合并保存到 `path_bruteforce_combined.txt`
- ✅ 统计可访问路径数量

---
